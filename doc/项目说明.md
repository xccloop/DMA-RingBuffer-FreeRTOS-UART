# STM32 DMA + RingBuffer + FreeRTOS UART 通信框架

## 1. 项目概述

本项目是一套基于 STM32 平台的 **DMA + 环形缓冲区 + FreeRTOS** 的 UART 串口通信框架。通过将 DMA 的高效数据传输、环形缓冲区的无锁读写、以及 FreeRTOS 的任务调度与同步机制相结合，实现**高吞吐、低延迟、非阻塞**的串口收发方案。

### 核心目标

- **零 CPU 等待发送**：发送数据时，CPU 只需将数据写入 RingBuffer，DMA 自动完成传输，任务立即返回。
- **无丢失接收**：DMA 持续将 UART 数据搬运至 RingBuffer，上层任务按需读取，不丢字节。
- **线程安全**：基于 FreeRTOS 信号量/互斥锁保护临界区，支持多任务并发访问同一 UART 端口。
- **可裁剪可移植**：通过配置宏选择 DMA 模式、缓冲区大小、流控策略，适配 STM32F1/F4/H7 等系列。

## 3. 模块说明

### 3.1 RingBuffer（环形缓冲区）

| 项目 | 说明 |
|------|------|
| 文件 | `ringbuffer/ringbuffer.h`, `ringbuffer/ringbuffer.c` |
| 实现 | 基于 `head`/`tail` 双指针的经典环形缓冲区 |
| 容量 | 编译期静态指定，必须是 2 的幂（利用位掩码取模，避免除法） |
| 线程模型 | **SPSC (Single Producer, Single Consumer)** —— DMA ISR 为生产者，用户 Task 为消费者（RX）；用户 Task 为生产者，DMA ISR 为消费者（TX） |
| 特性 | 无需锁（SPSC 场景下 head/tail 分别由两端独占修改），支持 `peek` 不消费读取，支持 `flush` 丢弃已读数据 |

**数据结构：**

```c
typedef struct {
    volatile uint32_t head;    // 写指针（生产者修改）
    volatile uint32_t tail;    // 读指针（消费者修改）
    uint8_t         *buffer;   // 数据存储区
    uint32_t         mask;     // 容量掩码 = capacity - 1
    uint32_t         capacity; // 缓冲区总容量（2 的幂）
} ringbuffer_t;
```

**核心 API：**

```c
void     rb_init(ringbuffer_t *rb, uint8_t *buf, uint32_t capacity);
uint32_t rb_put(ringbuffer_t *rb, const uint8_t *data, uint32_t len);
uint32_t rb_get(ringbuffer_t *rb, uint8_t *dst, uint32_t len);
uint32_t rb_peek(const ringbuffer_t *rb, uint8_t *dst, uint32_t len);
uint32_t rb_available(const ringbuffer_t *rb);
uint32_t rb_free_space(const ringbuffer_t *rb);
bool     rb_is_empty(const ringbuffer_t *rb);
bool     rb_is_full(const ringbuffer_t *rb);
void     rb_flush(ringbuffer_t *rb);
```

### 3.2 DMA UART 驱动层

| 项目 | 说明 |
|------|------|
| 文件 | `driver/uart_dma.h`, `driver/uart_dma.c` |
| 依赖 | STM32 HAL 库, FreeRTOS CMSIS_V2 |

#### TX 发送流程

1. 应用调用 `uart_send()` → 检查发送 RingBuffer 是否有空闲空间，将数据 `rb_put` 写入 TX RingBuffer。
2. 若 DMA 空闲，计算 RingBuffer 中从 `tail` 到 `head` 的**连续可传输段**，启动一次 DMA 传输。
3. **环形缓冲区 DMA 分段传输**：由于 RingBuffer 数据可能回绕（tail > head），分两种情况：
   - **未回绕**（tail < head）：单一 DMA 传输覆盖 `[tail, head)` 整段连续数据。
   - **已回绕**（tail > head）：先传输 `[tail, capacity)` 段，TC 中断中检查是否有第二段 `[0, head)`，若有则启动第二次 DMA 传输（链式）。
4. DMA TC 中断中更新 RingBuffer 读指针（tail），检查 RingBuffer 是否有新数据到达（head 是否再次前进），有则继续启动下一次 DMA。
5. 当 RingBuffer 为空且 DMA 停止时，通过 TaskNotify / Semaphore 通知等待任务。

```
用户 Task                          DMA ISR
   │                                 │
   ├─ uart_send(buf, len)            │
   │  ├─ rb_put(tx_rb, buf)          │
   │  └─ 若 DMA 空闲, 启动传输 ─────►├─ DMA TC 中断 (第1段完成)
   │                                 │  ├─ 更新 tail
   │                                 │  ├─ 检查回绕: head < tail?
   │                                 │  │   ├─ 是: 启动第2段 DMA [0, head)
   │                                 │  │   └─ 否: 检查 head > tail?
   │                                 │  │       ├─ 是: 有新数据, 继续传输
   │                                 │  │       └─ 否: 发送完成, 通知 Task
   │  ◄── xTaskNotify / Semaphore ────│
   ▼                                 ▼
```

#### RX 接收流程 (DMA 循环模式)

1. 初始化时启动 DMA RX 循环模式，指向 RX RingBuffer。
2. DMA 持续将 USART→DR 数据搬运至 RX RingBuffer。
3. DMA 半满/全满中断 → 更新 RingBuffer 写指针（head）。
4. **UART IDLE 中断** → 检测帧间隔，立即更新 head，通知接收任务有新数据到达。

```
DMA Controller              DMA ISR                   用户 Task
   │                          │                          │
   ├─ 循环搬运 USART→RB ────►├─ HT/TC 中断               │
   │                          │  └─ 更新 rb.head           │
   │                          │                            │
   │                          ├─ IDLE 中断                 │
   │                          │  └─ xTaskNotify(rx_task)  ├─ uart_recv(buf, len)
   │                          │                            │  ├─ rb_get(rx_rb, buf)
   │                          │                            │  └─ 返回实际字节数
   ▼                          ▼                            ▼
```

### 3.3 UART API 层

| 项目 | 说明 |
|------|------|
| 文件 | `api/uart_api.h`, `api/uart_api.c` |

提供面向应用任务的高级接口，屏蔽底层 DMA 和 RingBuffer 细节。

```c
// ---- 初始化 ----
void uart_init(uint8_t port, uint32_t baudrate,
               uint32_t tx_buf_size, uint32_t rx_buf_size);

// ---- 阻塞发送 (带超时) ----
int32_t uart_send(uint8_t port, const uint8_t *data,
                  uint32_t len, TickType_t timeout);

// ---- 阻塞接收 (带超时) ----
int32_t uart_recv(uint8_t port, uint8_t *buf,
                  uint32_t len, TickType_t timeout);

// ---- 非阻塞发送（立即返回，失败返回 -1）----
int32_t uart_send_async(uint8_t port, const uint8_t *data, uint32_t len);

// ---- 非阻塞接收 (返回当前 RingBuffer 中已有字节数) ----
int32_t uart_recv_async(uint8_t port, uint8_t *buf, uint32_t len);

// ---- 格式化输出 ----
int32_t uart_printf(uint8_t port, const char *fmt, ...);

// ---- 刷新缓冲 ----
void uart_tx_flush(uint8_t port);
void uart_rx_flush(uint8_t port);

// ---- 注册接收回调 (可选的按行/按帧解析) ----
void uart_set_rx_callback(uint8_t port, uart_rx_cb_t cb, void *arg);

// ---- 等待接收事件 (配合回调使用, 阻塞等待并驱动回调) ----
void uart_yield(uint8_t port, TickType_t timeout);
```

> **线程安全说明**：`uart_send` / `uart_printf` 内部通过 FreeRTOS Mutex 保护同一端口的 TX RingBuffer，多任务并发写同一端口不会出现数据交错。不同端口之间独立互斥，互不影响。

## 4. 关键技术细节

### 4.1 为什么用 SPSC 环形缓冲区

- DMA ISR 只修改 `head`，用户 Task 只修改 `tail`，不存在并发写同一变量的情况。
- 不需要互斥锁，不会出现禁用中断等待锁的情况，保证 ISR 的确定性延迟。
- **Cortex-M 单核可见性**：对于 ISR 与 Task 之间的单写者场景，`volatile` 已足够阻止编译器错误优化读写顺序。Cortex-M3/M4/M7 的单核内存模型保证 ISR 返回后 Task 可见所有 ISR 内写入，无需额外 `__DSB()`（仅在 Memory-to-Memory DMA 场景下才需要数据同步屏障）。

### 4.2 链式 DMA 传输与 RingBuffer 回绕处理

TX 端关键挑战：RingBuffer 写入数据超过缓冲区末尾时会回绕，数据分裂为两段不连续的物理地址。DMA 单次传输只能处理连续地址，因此需要**分段传输**：

1. **snapshot head**：启动传输前快照当前 `head`，防止传输期间新写入数据干扰边界计算。
2. **第 1 段**：启动 DMA 传输 `[tail, min(snapshot_head, capacity))` 段。
3. **TC 中断 → 第 2 段**：若 `snapshot_head < tail`（回绕），第 1 段完成时 TC 中断启动第 2 段 `[0, snapshot_head)`。
4. **持续检查**：每段 TC 中断完成后再次对比 `tail` 与当前 `head`，发现新数据则继续启动传输；RingBuffer 为空则停止 DMA 并通知 Task。

此方案利用 DMA 的 TC 中断自动串联多段物理内存，避免了从 RingBuffer 额外拷贝到临时线性缓冲区的开销。

### 4.3 UART IDLE 中断与帧检测

STM32 UART 在收到一个完整帧后的空闲时间内会产生 IDLE 中断。利用此特性：

- 不需要知道对方发送数据的长度。
- 不依赖字节间超时定时器。
- **实现要点**：IDLE 中断中通过 `__HAL_DMA_GET_COUNTER()` 读取 DMA 剩余传输计数，反算已传输字节数，精确更新 head。注意先读 NDTR 再清 IDLE 标志位，避免竞态。
- 硬件上，IDLE 帧间隔通常为 1 个字节时间，9600bps 下约 1ms，1Mbps 下仅 10μs，需确保 ISR 执行时间在此窗口内。

### 4.4 RX 溢出处理策略

DMA 循环写入 RingBuffer 时，若上层 Task 消费速度低于接收速率，DMA 会回绕覆盖尚未消费的数据（静默丢数据）。处理策略：

- **检测**：DMA TC/HT 中断更新 head 时，若 `head` 越过 `tail`（即新写入量 > RingBuffer 剩余空间），记录溢出标志 `rx_overrun` 并标记实际可读字节数为 `capacity`。
- **丢旧策略**：溢出不阻塞 DMA（DMA 循环模式不可暂停），丢弃最旧的未消费数据，`tail` 跳到 `head - capacity` 位置，保证最新数据可读。
- **通知**：溢出事件通过回调或状态标志上报应用层，可选触发流控。

### 4.5 发送完成通知策略

- 当 RingBuffer 为空且 DMA 停止时，通过 `TaskNotify` 通知等待任务。
- 若使用 BinarySemaphore 模式（`UART_USE_TASK_NOTIFY = 0`），需注意 Semaphore 的计数语义——只通知"有空间"，具体可写字节数由 `rb_free_space()` 确定。

### 4.6 内存管理

- 所有缓冲区在编译期静态分配（`static uint8_t tx_buf[...]`），避免动态分配碎片化。
- 配置宏控制整体内存占用，便于在资源受限的 MCU 上调整。

```c

void uart_port_init(uint8_t port, uint32_t baudrate);
void uart_port_deinit(uint8_t port);

// ---- DMA 操作 ----
void uart_port_dma_tx_start(uint8_t port, const uint8_t *data, uint32_t len);
```

## 6. 使用示例

### 6.1 初始化和回显

```c
#include "uart_api.h"

void echo_task(void *param)
{
    uint8_t buf[128];

    uart_init(1, 115200, 512, 1024);  // UART1, 波特率 115200

    while (1) {
        // 阻塞等待接收数据（超时 1 秒）
        int32_t n = uart_recv(1, buf, sizeof(buf), pdMS_TO_TICKS(1000));
        if (n > 0) {
            uart_send(1, buf, n, pdMS_TO_TICKS(100)); // 原样发回
        }
    }
}
```

### 6.2 帧解析（基于 IDLE + 回调）

```c
void on_frame_received(uint8_t port, const uint8_t *data, uint32_t len, void *arg)
{
    // 收到完整一帧（由 UART IDLE 自动分包）
    process_packet(data, len);
}

void parser_task(void *param)
{
    uart_init(2, 9600, 256, 512);
    uart_set_rx_callback(2, on_frame_received, NULL);

    while (1) {
        uart_yield(2, pdMS_TO_TICKS(100)); // 让出 CPU，等待事件
    }
}
```

### 6.3 格式化日志输出

```c
void logger_task(void *param)
{
    uart_init(3, 115200, 1024, 128);

    uart_printf(3, "System booted, tick = %lu\r\n", xTaskGetTickCount());
    uart_printf(3, "CPU freq: %lu MHz\r\n", SystemCoreClock / 1000000);
}
```

## 7. 性能指标 (预估, STM32F407 @168MHz)

| 指标 | 数值 | 备注 |
|------|------|------|
| 最大波特率 (DMA) | 6 Mbps | 受外设时钟和 GPIO 翻转速率限制 |
| TX 吞吐量 | ~5 Mbps | RingBuffer memcpy 非瓶颈 (168MHz 总线带宽 > 50MB/s)，实际受限因素为 UART 波特率 |
| RX 吞吐量 | ~5 Mbps | 同上，持续满速率接收要求在 5Mbps 时 Task 每 1.6ms 至少消费 1KB |
| 单字节发送延迟 | < 2 μs | DMA 空闲且 RingBuffer 为空时，直接启动一次传输 |
| RAM 占用 (3 端口, 默认配置) | ~8 KB | 含每端口 TX/RX 缓冲区 + 控制结构体 |
| Flash 占用 | ~4 KB | 代码段，不含 HAL 库本身 |
| ISR 最大执行时间 | < 3 μs | TC/HT 中断：仅指针更新 + 检查是否需要下次传输 |
